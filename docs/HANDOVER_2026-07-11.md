## 五、環境備忘（實戰經驗，下個 session 慳時間）

- **Sandbox 每個 bash call 完結會 kill 晒 child process**（`--die-with-parent`），`nohup`/`setsid` 都留唔低背景 process——vitest 一定要喺單一 call 45 秒內跑完，**分 chunk 跑**（3–7 個 test 檔一 chunk 最穩陣，sandbox 慢嗰陣 3 個都會 timeout，timeout 完要 sleep 幾秒等 RPC 釋放先開得新 call）。
- `github_push.py` 係成棵 tree 一個 commit——所以今次所有改動係**一個 commit push**（每一步已經獨立驗證綠，詳見上文），做唔到真・分 commit。想 per-step commit 就要每步完即 push 一次。
- push 前用 dry-run（照抄 script 邏輯只印 diff 唔寫）睇清楚會 upload／delete 啲乜，今次就係咁樣捉到 BUILD_SPEC 會被誤刪。
